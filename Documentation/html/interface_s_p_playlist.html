Move item(s) to another location in the list.

All indexes are given relative to the state of the item order before the move is executed. Therefore, you don't* need to adjust the destination index to take into account items that will be moved from above it.

Warning
This operation can fail, for example if you give invalid indexes. Please make sure you check the result of this method.
Parameters
indexesThe indexes of the items to move.
newLocationThe index the items should be moved to.

- (BOOL) collaborative

readnonatomicassign

Returns YES if the playlist has changes not yet recognised by the Spotify servers, otherwise NO.

- (SPImage *) image
readnonatomicstrong

Returns the custom image for the playlist, or nil if the playlist hasn't loaded yet or it doesn't have a custom image.

- (NSArray *) items
readatomiccopy

Returns an array of SPPlaylistItem objects representing playlist's item order.

- (BOOL) loaded
readnonatomicassign

Returns YES if the playlist has finished loading and all data is available.

- (BOOL) markedForOfflinePlayback
readwritenonatomicassign

Returns YES if the playlist is marked for offline playback.

- (NSString *) name
readwritenonatomiccopy

Returns the name of the playlist, or nil if the playlist hasn't loaded yet.

- (float) offlineDownloadProgress
readnonatomicassign

Returns the download progress of the playlist (between 0 and 1) is it is marked for offline sync.

- (sp_playlist_offline_status) offlineStatus
readnonatomicassign

Returns the offline status of the playlist. Possible values:

SP_PLAYLIST_OFFLINE_STATUS_NO Playlist is not offline enabled.

SP_PLAYLIST_OFFLINE_STATUS_YES Playlist is synchronized to local storage.

SP_PLAYLIST_OFFLINE_STATUS_DOWNLOADING This playlist is currently downloading. Only one playlist can be in this state any given time.

SP_PLAYLIST_OFFLINE_STATUS_WAITING Playlist is queued for download.

- (SPUser *) owner
readnonatomicstrong

Returns the owner of the playlist, or nil if the playlist hasn't loaded yet.

- (sp_playlist *) playlist
readnonatomicassign

Returns the opaque structure used by the C LibSpotify API.

Warning
This method must be called on the libSpotify thread. See the "Threading" section of the library's readme for more information.
This should only be used if you plan to directly use the C LibSpotify API. The behaviour of CocoaLibSpotify is undefined if you use the C API directly on items that have CocoaLibSpotify objects associated with them.
- (NSString *) playlistDescription
readnonatomiccopy

Returns the custom description for the playlist, or nil if the playlist hasn't loaded yet or it doesn't have a custom description.

- (__unsafe_unretained SPSession *) session
readnonatomicassign

Returns the session object the playlist is loaded in.

- (NSURL *) spotifyURL
readnonatomiccopy

Returns the Spotify URI of the playlist profile, for example: spotify:user:sarnesjo:playlist:3p2c7mmML3fIUh5fcZ8Hcq

- (NSArray *) subscribers
readnonatomicstrong

Returns the subscribers to the playlist as an array of Spotify usernames.

- (BOOL) updating
readnonatomicassign

Returns YES if the playlist is being updated, otherwise NO.

Typically, you should delay UI updates while this property is set to YES.
