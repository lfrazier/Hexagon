Detailed Description

Delegate callbacks from SPSession specifically to do with delivering audio to the audio device. This protocol replaces the audio delivery method in SPSessionPlaybackDelegate.

Method Documentation

- (NSInteger) session: (id< SPSessionPlaybackProvider >)  aSession
shouldDeliverAudioFrames: (const void *)  audioFrames
ofCount: (NSInteger)  frameCount
streamDescription: (AudioStreamBasicDescription)  audioDescription 

Called when audio data has been decompressed and should be pushed to the audio buffers.

See the SimplePlayback sample project for an example of how to implement audio playback.

Warning
This function is called from an internal session thread - you need to have proper synchronization!
If this method is called with a frameCount of 0, an "audio discontinuity" has occurred - for example, the user has seeked playback to another part of the track. You should clear audio buffers and prepare for new audio.
This function must never block. If your output buffers are full you must return 0 to signal that the library should retry delivery in a short while.
Parameters
aSessionThe session providing the audio data.
audioFramesA buffer containing the audio data.
frameCountThe number of frames in the buffer.
audioDescriptionAn AudioStreamBasicDescription containing information about the audio format.
Returns
Number of frames consumed. This value can be used to rate limit the output from the library if your output buffers are saturated. Delivery will be retried in about 100ms.
